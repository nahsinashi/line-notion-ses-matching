## 1. 日次運用

### 📝 データ登録タスク

#### 1.1 案件・要員の登録（随時）

**担当者**: 営業メンバー全員

**作業内容**:
- 新規案件が来たら「🧾 案件DB」に登録
- 新規要員が来たら「👤 要員DB」に登録
- マッチング候補があれば「📋 提案DB」に登録

**登録ルール**:
- **案件DB**: ステータスは「未処理」で登録
- **要員DB**: ステータスは「未処理」で登録
- **提案DB**: ステータスは「候補」で登録
- 担当者を必ず設定

#### 1.2 ステータス更新（随時）

**担当者**: 各案件・要員の担当者

**更新タイミング**:
- 提案を送ったら → 「提案中」
- 面談が設定されたら → 「面談」
- 面談日を「面談設定日」に記入
- 結果待ちになったら → 「結果待ち」
- 決定したら → 「決定」
- 見送り・辞退になったら → 該当ステータスに変更

**注意事項**:
- ステータス変更は**即座に**行う（分析の精度に影響）
- メモ欄に経緯を簡潔に記入
- 日付フィールドは必ず入力

## 2. 週次運用

### 📊 週次レポート生成

**実施日**: 毎週月曜日 午前中
**担当者**: 管理者 or 営業リーダー

#### 2.1 事前準備（月曜日 9:00-9:30）

1. **先週のデータ確認**
   - 提案DB: ステータス更新漏れがないか確認
   - 要員DB: 終了案件の記録漏れがないか確認
   - 案件DB: 決定・終了の記録漏れがないか確認

2. **営業コスト管理DBに入力**
   - 前週（月〜日）の累積稼働時間を入力
   - 週間予算時間を入力（通常40h）
   - 時間単価を入力（通常¥5,000/h）
   - 備考があれば記入

#### 2.2 レポート生成（月曜日 9:30-10:00）

**Claude Code を使用する場合**:
```bash
# ターミナルで実行
cd C:\Users\user\claude-workspace\_skills\ses-analysis-skill
claude-code

# Claude Code 内で
週次レポートを生成してください
```

**Claude.ai アプリを使用する場合**:
```
週次レポートを生成してください
```

#### 2.3 レポート確認（月曜日 10:00-10:30）

1. **⚠️ 期限超過アラートの確認**
   - 提案DB（1週間以上経過）: 即座にフォローアップ
   - 要員DB（2週間以上経過）: ステータス確認・更新
   - 案件DB（2週間以上経過）: ステータス確認・更新

2. **📊 数値の確認**
   - 営業コスト予算消化率: 95-100% が理想
   - 新規登録数: 前週比でトレンド確認
   - 決定件数: 目標達成度を確認

3. **📈 異常値のチェック**
   - 見送り・辞退が多い → 原因分析
   - 新規登録が少ない → 営業活動の見直し
   - 期限超過が多い → プロセス改善

#### 2.4 チーム共有（月曜日 10:30-11:00）

- Notionの「最新週次レポート」をチームに共有
- 期限超過アラートの対応担当を決定
- 今週の重点アクションを確認

## 3. 月次運用

### 📈 月次レポート生成

**実施日**: 毎月1日 午前中
**担当者**: 管理者

#### 3.1 事前準備（1日 9:00-9:30）

1. **先月のデータ最終確認**
   - 全データベースのステータスを確認
   - 決定・終了の最終記録を確認

2. **営業コスト管理DBの確認**
   - 先月の全週（4-5週分）のデータが揃っているか確認

#### 3.2 レポート生成（1日 9:30-10:30）

**Claude Code または Claude.ai で**:
```
月次レポートを生成してください
```

#### 3.3 レポート分析（1日 10:30-12:00）

1. **月次KPIの確認**
   - 提案決定率: 目標 30% 以上
   - 営業コスト効率: コストあたり粗利 0.5 以上
   - 見送り率: 40% 以下が望ましい

2. **進行中案件の確認**
   - 面談・結果待ちの案件: 優先フォロー
   - オファー中の要員: 結果確認
   - 面談調整中の案件: 決定促進

3. **トレンド分析**
   - 3ヶ月トレンド: 改善傾向か悪化傾向か
   - 前月比: 各指標の増減を分析

#### 3.4 経営報告・方針決定（1日 午後）

- 月次レポートを経営層に報告
- 来月の重点施策を決定
- 予算・目標の見直し

## 4. データ登録ルール

### 📝 必須ルール

#### 4.1 案件DB 登録ルール

| 項目 | ルール |
|------|--------|
| **案件名** | わかりやすく簡潔に（例: 【X社】ECサイト構築） |
| **担当** | 必須（髙梨/坂本/堀江） |
| **ステータス** | 登録時は「未処理」 |
| **原文** | パートナーからの情報をそのまま貼り付け |
| **案件元企業** | 必須 |
| **案件回収日** | 自動入力（変更不可） |

#### 4.2 要員DB 登録ルール

| 項目 | ルール |
|------|--------|
| **要員名** | イニシャル_最寄駅（例: K.S_渋谷） |
| **担当** | 必須（髙梨/坂本/堀江） |
| **ステータス** | 登録時は「未処理」 |
| **原文** | パートナーからの情報をそのまま貼り付け |
| **要員元企業** | 必須 |
| **要員回収日** | 自動入力（変更不可） |

#### 4.3 提案DB 登録ルール

| 項目 | ルール |
|------|--------|
| **提案名** | 自動生成（案件名＋要員名） |
| **案件DB** | リレーションで案件を選択（必須） |
| **要員DB** | リレーションで要員を選択（必須） |
| **ステータス** | 登録時は「候補」 |
| **提案日** | 実際に提案を送った日付を入力 |
| **メモ** | 経緯・アクションを記録 |

#### 4.4 営業コスト管理DB 登録ルール

| 項目 | ルール |
|------|--------|
| **対象週** | 「YYYY年MM月第X週（M/D-M/D）」形式 |
| **週開始日** | その週の月曜日 |
| **週終了日** | その週の日曜日 |
| **累積稼働時間** | チーム全体の実績時間（例: 38.0） |
| **週間予算時間** | 通常40h |
| **時間単価** | 通常¥5,000/h |

### ⚠️ 禁止事項

- ❌ ステータスの飛ばし（未処理→決定など）
- ❌ 日付フィールドの空欄放置
- ❌ 担当者未設定
- ❌ 重複登録
- ❌ 古いデータの削除（履歴として残す）

## 6. トラブルシューティング

### 🔧 よくある問題と対処法

#### 問題1: レポート生成時にNotionエラー

**症状**: "Notion API接続エラー" が表示される

**原因**:
- Notion統合のアクセス権限が不足
- APIキーの期限切れ
- ネットワーク接続の問題

**対処法**:
1. Notion統合の接続を確認
2. 各DBへのアクセス権限を再確認
3. ネットワーク接続を確認
4. Claude Code/Claude.ai を再起動

#### 問題2: データが取得できない

**症状**: "データが0件です" と表示される

**原因**:
- 対象期間にデータが存在しない
- フィルター条件が厳しすぎる
- データベースIDが間違っている

**対処法**:
1. Notion上で手動でデータを確認
2. 対象期間を変更してみる
3. データベースURLを再確認

#### 問題3: 期限超過アラートが多すぎる

**症状**: 期限超過が10件以上

**原因**:
- ステータス更新の遅れ
- 案件進行の停滞

**対処法**:
1. 各案件の担当者に状況確認
2. ステータス更新を依頼
3. 停滞している案件は「終了」に変更

#### 問題4: 粗利見込が計算できない

**症状**: "計算不可" と表示される

**原因**:
- 案件単価または要員希望単価が未入力
- リレーションが設定されていない

**対処法**:
1. 提案DBで該当レコードを確認
2. 案件DBと要員DBのリレーションを確認
3. 単価フィールドを入力
